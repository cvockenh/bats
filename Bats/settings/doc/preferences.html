Preferences

In Prefereces you can make some general settings.

Not up-to-date anymore!

Calc

The calculation preferences are very important! They should be checked for every evaluation!

Isobar

Isobar: If there is an interference of an isobar on the ratioisotope, the isobar can be selected here.

Isobar factor: The measured isobar is multiplied with this factor and subtracted from the radioisotope.

Isobar error: In addition to the error an additional relative error of the correction itself can be given here.

Blank

Blank scatter: Experianced blank scatter of the blank (this is added to the statistical error of the mean of mean of the blanks)

Nominal ratio (pmC)

ra_fin: The nominal ratio for the "virtual" standard can be given here. The measured standard ratios (shown in the standard table with plot) are given normalised to this virtual value.

Current settings

Offset: An offset for the measured currents can be subtracted here.

rel. error: The relative (estimated) error of the current measurement can be given here.

abs. error: The absolute (estimated) error of the current measurement can be given here.

Calc

always recalculate: If active the data should be recalculate whenever a change to the data is done (moderate slowdown).

Poisson distribution: If true, not the square root from the counts is taken as an error, but the upper limit of a poisson distribution.

Fractionation

13/12 correction on 14/12: If true, a correction for fractionation (from chemistry or instrument) on the 14/12 ratio of the standard is done for the standard mean calculation (should reduce error of standard). Only applicable, if it is a mass fractionation!

File

Save

autosave std/bl plot: Automatically saves standard and blank plots to a .png file (moderate slowdown). Active for every recalculation!

autosave output file: Automatically saves output to .xhtml file (large slowdown). Active for every recalculation!

output without runs: Saves only sample data without runs. If inactive also runs are saved.

page break after... : A page break is inserted after given number of lines in the output file. Each time, a new header is inserted (the real page-break should be done, when prited).

add correction to output: Adds corrections at the end of the file.

autosave bats file: Automatically saves a .bats-file after every recalculation with all data and corrections.

DB

Oracle import

virtual: Data import from database (DB) is from virtual tables (recalculated at the time of the import)

cycle import: Instead of runs, cycles can be imported as well. Attention, this can be a lot of data! Therefore the number of cycles taken for one run at the time of the import should be high enough!
